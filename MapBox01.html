Baxter Creak Watershed Alliance

Exploring MapBox as a solution for the Baxter Creek Water Shed Alliance Website


Problem:

There are no good maps about the Baxter Creek Watershed. As a result, visitors to the area are not aware of the Baxter Creek Watershed and it's ecological or natural features.They may get lost whilst using the trails and miss key points of interest.


Solution:

To create a thematic map to display and educate the general public on the Baxter Creek Watershed, using MapBox.


What is MapBox?

MapBox is an online tool that gives people access to nagivation tools. Technology behind many apps that help people get to where they need to be. Thousands of companies utilize MapBox services in aid of navigational software in cars and smartphones. There are over 600 million monthly active users that trust their ability to run their services. This is a global service delivering to customers in farming, manufacturing, ecommerce and more industries.


Why use MapBox?

MapBox delivers results for any type of service needed. They are constantly updating their services to keep up with industry requirements. There are many options of Mapbox to fully supply any services needed of a client. The clientelle ranges from BMW to National Geographic to CNN.


Solution Steps:

Below are the steps we took to create the map featured on this page:


Step 1:

A group Github repository was created for Group4. All files used in this website are hosted there.


Step 2: Quick Start Guide - MapBox

The group created MapBox01.html using the Quick Start Guide featured. This guide used the mapbox/streets-v11 tiles from Mapboxâ€™s Static Tiles API. In order to use this, we were asked to use an access token. Thus, we requested the public default access token to use the tile layer from MapBox. We experienced difficulties using this token and accessing the tile layer.


Step 3: Simple Leaflet Examples - Github

Alternative examples in Github were turned to understand how to integrate tiles into the map we were creating. The following examples were used to understand how different plugins looked once deployed from Github. Even though they were slightly complex to follow along with at times, it helped in understanding the basics of using Leaflet. These examples were created by by Klaus Stein and Stephanie Saephan in Github.

insert tile layer 1 here


Step 4:Implement Tiles

This webpage created by Andy Woodruff, Ryan Mullins and Cristen Jones for Maptime Boston featured a rather simple tutorial to Leaflet which the group eneded up using. We used their example to integrate the tile layer: OpenStreetMap.Mapnik from here into our map using JQuery. There was difficulty in downloading JQuery v3.6.0 from jquery.com in Chrome, Mozilla and Edge for some time for reasons unknown. However, the JQuery Javascript library was downloaded and uploaded to the repository.

insert tile layer 1 here


Step 5:Converted MVT_Trails_Feature shape file to GeoJSON file

The MVT_Trails_Features shape file was downloaded from ArcGIS Online and converted to GeoJSON using ArcGISPro using the Features to GeoJSON conversion tool. This was uploaded to the Group 4 repository and accessed/added to the current map using Javascript.

insert screen shot


Step 6: Created points Fishing Spots using geojson.io

The group used geojson.io to create hypothetical fishing points in the Baxter Creek Water Shed. This GeoJSON file was saved to the Group 4 Github repository. These points were edited to represent fishing icons and added to the current map using Javascript.

insert screen shot


Step 7: Map formatting and Sizing Difficuties

When resizing the map, there seemed to be conflict with the custom css file. Percentage sizings did not work whilst fixed pixel sizings did.

insert screen shot


Step 6: Iframe

We used iframe to put one interactive map html into another webpage. We did this to better control the dimension of the map frame.
